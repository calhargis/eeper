# Manual testing procedures

The [implementation plan](../IMPLEMENTATION_PLAN.md) labels each testing
criterion `[AUTO]` (verified in CI) or `[MANUAL]` (requires a human, physical
hardware, or perceptual judgement). Written procedures for the `[MANUAL]`
criteria live here, one file per procedure, added as each milestone reaches it.

When a manual procedure is performed for a release candidate, record the date
and tester initials in [PROGRESS.md](../PROGRESS.md) next to the criterion
(e.g. `✔ 2026-08-14 JD`).

lands with the Compose/TLS work._
